使用hugo与github搭建博客

2019-08-14
软件安装
约 1777 字 预计阅读 4 分钟

今天使用hugo来搭建博客，猜了很多的坑，特来写博客记录一下。

有关Hugo的介绍

它是go语言实现的一个博客生成器，世界上最快的博客生成器，之前用JS配置的hexo可以说很难用。

hugo安装

这里就只讲一下windows用户怎么安装hugo吧（只用过windows）

  1. 去Hugo releases页面下载对应的版本，根据自己的系统选择32位或者64位

  2. 解压出下载的安装包，把解压出来的hugo.exe这个文件放到自己装软件的目录下，例如我的就是：F:/软件/hugo/hugo.exe

  3. 然后把hugo.exe这个文件加到PATH下，例如：F:/软件/hugo/加到PATH下，这一步很重要，不然无法在其他目录使用。顺便提一下win10加PATH的方法，右键点击我的电脑，点击属性，然后点击高级系统设置，再点击右下角的环境变量，找到PATH，点进去新建path，把刚刚的路径（F:/软件/hugo/hugo.exe）加进去，这样你就可以在任何目录使用了。

  4. 然后重启终端，输入指令 hugo version ，就会出现对应的版本号。例如我的就是：Hugo Static Site Generator v0.58.3-4AAC02D4 windows/amd64 BuildDate: 2019-09-19T15:29:19Z。

这个时候你的hugo就安装成功了。

使用hugo快速搭建博客

这里推荐进入官方网站,点击Quick Start

  • 官方文档上的step1咱们已经做了，直接进入step2，step2的其实就是在你所在的目录建立一个文件夹，并且初始化一些配置，指令：hugo new site quickstart（后面这个quickstart是你文件夹的名称，自定义就好，建好之后里面会有一些初始化的文件）。

  • step3：首先进入刚刚创建的文件夹，cd quickstart（文件夹名称）,然后git init （这一步是待会与github的仓库连接起来的初始化），然后运行 git submodule add https://github.com/budparr/gohugo-theme-ananke.git themes/ananke 这一个指令其实是在themes目录下创建一个ananke目录，并且把这个Git仓库的主题添加进去，然后运行 echo ‘theme = “ananke”’ >> config.toml 把这句话添加到toml这个配置文件中，可以引用刚刚下载的主题。

  • step4：这个时候其实是可以写一写文章了，运行 hugo new posts/my-first-post.md，这句指令会在当前（quickstart）目录下的content目录下，创建一个posts目录，并且在这个目录下my-first-post.md 生成这个文件。打开这个文件，如下：

上面的draft是草稿的意思，如果为ture，它是不能够提交的，在提交的时候千万要记得改成false。

  • step5：运行 hugo server -D，这个指令会用服务器产生一个路径 http://localhost:1313/你可以预览。

  • step6：配置config.toml这个文件，这个是你的博客的一下基础设置，例如，刚刚以上步骤之后生成的：

    baseURL = "http://example.org/"
languageCode = "en-us"
title = "My New Hugo Site"
theme = "ananke"

你可以自定义修改语言，主题（前提是下载了这个主题，后面会讲主题下载安装使用）等。

  • step7：这个时候到了最后一步，运行hugo指令，你会在目录下快速生成一个public的文件夹，这个文件夹会用来与你的github仓库绑定。

到这里，hugo已经可以生成博客了。

hugo与github的结合使用

一开始看官方教程的时候没有这一步，后面看了很多文章，踩了很多坑才成功的。

  • 首先在你最开始那个目录下创建一个.gitignore的文件，并且在里面写上/public/，这个是为了与public外面的目录区分，形成两个仓库。
  • 然后运行cd public ，进入public这个文件夹，运行git init ，git add . ，git commit - ' 第一篇博客 post' 。
  • 然后打开你的github，新建一个Git仓库，注意：这个仓库的名称必须为你的用户名.github.io,然后运行git remote add origingit@github.com:你的用户名/你的用户名.github.io.git，再然后git push -u origin master,刷新浏览器，你会看到你的git仓库已经有刚刚上传的东西了，打开你的用户名.github.io这个网站可以在网上预览你写的文章啦~~

后面每次写博客的时候，现在content/post的下面新建md文件，写完记得把草稿改为false，然后hugo一下，进入public目录，重复git add . ,git commit git push 这些指令就行。
