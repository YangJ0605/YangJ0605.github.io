按值传递

2019-10-12
JS深入
约 652 字 预计阅读 2 分钟

定义

在《JavaScript高级程序设计》第三版 4.1.3，讲到传递参数：

ECMAScript中所有函数的参数都是按值传递的。

什么是按值传递呢？

也就是说，把函数外部的值复制给函数内部的参数，就和把值从一个变量复制到另一个变量一样。

按值传递

例子1：

var value = 1;
function foo(v) {
    console.log(v) //1
    v = 2;
    console.log(v); //2
}
foo(value);
console.log(value) // 1

在函数未执行到v=2时，此时是把value的值传给V，所以此时v的值为1，当运行到v=2时候，此时把2赋值给v，value从头到尾没有变过。

内存分析：

改变前：

栈内存 堆内存
value 1
v 1

改变后：

栈内存 堆内存
value 1
v 2

var obj = {
value: 1
};
function foo(o) {
    console.log(o) //{value:1}
    o.value = 2;
    console.log(o.value); //2
}
foo(obj);
console.log(obj.value) // 2

obj中存放的是对象的地址，一开始把obj的地址传给o，此时它们两个存放着一样的地址，都指向同一个对象，当运行到o.value=2的时候，因为它们两个指向相同的对象，所以obj的value也会跟着改变。

改变前：

obj o 地址 {value:1}

改变后：

栈内存 堆内存
obj o 地址 {value:2}

var obj = {
value: 1
};
function foo(o) {
    console.log(o) //{value:1}
    o = 2;
    console.log(o); //2
}
foo(obj);
console.log(obj.value) // 1

obj中存放的是对象的地址，一开始把obj的地址传给o，此时它们两个存放着一样的地址，都指向同一个对象，当运行到o=2的时候，此时o从原来存放地址改为存放2，而obj还是存放着原来的地址，所以obj.value还是1

改变前：

栈内存 堆内存
obj,o 地址 {value:1}

改变后：

栈内存 堆内存
obj 地址 {value:1}
o 2

无论基本类型传递还是引用传递其实都是值传递，基本类型就是传递基本类型的值，比如1，2，3这样，引用类型传递的是地址值
